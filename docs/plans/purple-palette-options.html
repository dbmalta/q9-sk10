ScoutKeeper — Purple Palette Exploration

Four candidate purples, each shown with a full tonal scale, contrast-checked usage, and the UI elements that matter most — primary button, toggle, active nav item, and soft surfaces. All options avoid copying WOSM's exact brand hex (~#582D8D) while staying recognisably Scout-adjacent.

Toggle light/dark at top-right. Fonts: Inter (UI) + Fraunces (display).

How to read each option: the 11-step scale runs from 50 (tint) to 950 (near-black). The buttons, toggles, and nav use 500 (or 600 in dark mode) as the accent, 50/100 as soft backgrounds, and 700/800 for hover/pressed states.

Quick comparison

Option Personality Best for Watch-out
A — Heritage Violet Traditional, institutional, warm Broad appeal; feels closest to a Scout crest without copying WOSM Could read "old-fashioned" to very modern parents
B — Modern Violet Tech-forward, clean Parents who live in modern apps Generic — used by Tailwind defaults, many SaaS apps
C — Warm Plum Soft, friendly, community Parent/member surfaces with illustration Admins may find it less "serious"
D — Deep Indigo Grown-up, quiet, desaturated Tenant branding overrides (neutral host) Least distinctively "Scout"

My recommendation

Option A (Heritage Violet) is the strongest standalone pick — it gives ScoutKeeper an identity that nods to the movement's heritage without copying it, and works well both as a bold primary (admins) and via soft 50/100 tints (parent-facing warmth). Options B and C are plausible; B is the riskiest because it's so widely used elsewhere.

A second strategy worth considering: ship Option A as the ScoutKeeper default, but design the whole system around a single --accent-* CSS custom property, so each tenant can override to their own colour via the Admin settings. That way the palette is never a limiting choice — it's a starting point.
