# Roles & Permissions

ScoutKeeper uses a role-based access control system. Each user is assigned one or more roles, and each role grants specific permissions.

## Default roles

ScoutKeeper ships with a set of default roles:

| Role | Access |
|------|--------|
| Administrator | Full access to all features |
| Leader | Members, Events, Achievements (own section) |
| Secretary | Members, Communications (all sections) |
| Read Only | View-only access to members and events |

## Creating a role

1. Go to **Admin → Roles & Permissions**
2. Click **Add Role**
3. Enter a name and select the permissions to grant
4. Save

## Assigning roles

Roles are assigned to user accounts, not member records. Go to a user's account in **Admin → Users & Logins** to assign a role.

## Scoped access

Roles can be scoped to a specific section or node. A leader scoped to "1st Scouts Cubs" can only see and edit members in that section.
