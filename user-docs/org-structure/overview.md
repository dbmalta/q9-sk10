# Organisation Structure Overview

ScoutKeeper models your Scout organisation as a hierarchy of nodes — districts, groups, sections, and any other levels your organisation uses.

## Concepts

**Nodes** are the building blocks of the hierarchy. Each node has:

- A name and type (e.g. Group, Section, District)
- A parent node (except the root)
- Optional contact information

**Teams** are cross-cutting groups that do not follow the hierarchy — useful for leadership teams or sub-committees.

## Setting up your hierarchy

Before adding members, create the hierarchy that reflects your organisation:

1. Go to **Organisation Structure**
2. Add your top-level node (e.g. the Group name)
3. Add child nodes for each section (e.g. Beavers, Cubs, Scouts)

Members are then assigned to a section node when their record is created.
